# Database Design

## Introduction

SmartHire AI uses **PostgreSQL** as its relational database management system (RDBMS). The database is designed to securely manage users, resumes, AI-generated resume analysis, interview sessions, interview questions, candidate answers, and AI-generated interview evaluation reports.

The design follows **Third Normal Form (3NF)** to minimize redundancy, maintain data integrity, and provide scalability for future recruitment features.

---

# Database Objectives

The database is designed to:

- Store user accounts securely.
- Support Role-Based Access Control (Candidate, Recruiter, Administrator).
- Manage uploaded resumes.
- Store extracted resume information generated by AI.
- Generate and store AI interview questions.
- Manage interview sessions.
- Store candidate interview answers.
- Store AI evaluation reports and interview scores.
- Maintain referential integrity using foreign keys.
- Support future recruitment workflows.

# Database Entities

## 1. Users

### Purpose

Stores information about all registered users.

### Attributes

| Attribute | Type | Description |
|------------|------|-------------|
| id | Integer (PK) | Unique user ID |
| full_name | String | User's full name |
| email | String | Unique email address |
| hashed_password | String | Encrypted password |
| role | Enum | Candidate / Recruiter / Admin |
| created_at | Timestamp | Account creation date |

### Relationships

- One User can upload many Resumes.
- One User can participate in many Interview Sessions.

---

## 2. Resumes

### Purpose

Stores uploaded resumes for candidates.

### Attributes

| Attribute | Type | Description |
|------------|------|-------------|
| id | Integer (PK) | Resume ID |
| owner_id | Integer (FK → Users.id) | Resume owner |
| filename | String | Original file name |
| file_path | String | Resume storage path |
| extracted_text | Text | Parsed PDF content |
| uploaded_at | Timestamp | Upload timestamp |

### Relationships

- Many Resumes belong to one User.
- One Resume has one Resume Analysis.
- One Resume can generate multiple Interview Questions.
- One Resume can have multiple Interview Sessions.

# Database Normalization

The SmartHire AI database follows **Third Normal Form (3NF)**.

### First Normal Form (1NF)

- Atomic values only.
- No repeating groups.

### Second Normal Form (2NF)

- Every non-key attribute depends on the entire primary key.

### Third Normal Form (3NF)

- No transitive dependencies.
- Foreign keys maintain relationships.
- Redundant information is eliminated.

# Data Integrity

The database maintains integrity using:

- Primary Keys
- Foreign Keys
- Unique Email Constraint
- NOT NULL Constraints
- Cascading Relationships
- SQLAlchemy ORM Validation
